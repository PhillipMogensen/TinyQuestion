# TinyQuestion

A native macOS overlay for asking quick questions to a locally-running [Ollama](https://ollama.com) model. Summon it with a global hotkey, type, get a streaming answer, dismiss. Single window, no history, no menu bar icon, no Dock icon.

## Requirements

- macOS 14 (Sonoma) or later
- [Ollama](https://ollama.com) running locally on `localhost:11434`, with at least one model pulled (`ollama pull phi4-mini` is a good starting point — fast and capable for short prompts)

## Install

1. Download the latest `TinyQuestion-X.Y.Z.dmg` from the [Releases page](https://github.com/PhillipMogensen/TinyQuestion/releases).
2. Open the DMG and drag `TinyQuestion.app` to `/Applications`.
3. **First launch** — because TinyQuestion is unsigned, macOS Gatekeeper will block it. There are two ways through:

   **Option A — right-click in Finder:** open `/Applications`, right-click `TinyQuestion.app`, choose **Open**, then click **Open** in the dialog. macOS remembers this choice.

   **Option B — terminal:** strip the quarantine attribute that the browser added to the download:
   ```sh
   xattr -dr com.apple.quarantine /Applications/TinyQuestion.app
   ```
   Then double-click as normal.

   On macOS Sequoia (15) or later you may instead need to go to **System Settings → Privacy & Security**, scroll to the bottom, and click **Open Anyway** next to the TinyQuestion entry.

4. Grant accessibility permission if prompted (only required if a future version uses input monitoring; the current version does not).

## Use

- **Summon:** `⌥⌘J` (default — change in settings).
- **Type a question** in the input bar; press `Return` to send. Answers stream in below the input.
- **Settings:** while the overlay is open, press `⌘,` or click the gear. From there, pick a model, edit the system prompt, or rebind the hotkey.
- **Dismiss:** press `Esc`. The conversation is cleared — TinyQuestion is intentionally ephemeral.
- The overlay does not appear in the Dock or app switcher; it survives `⌘-Tab`.

## Build from source

```sh
brew install xcodegen create-dmg     # one-time tooling
xcodegen generate
xcodebuild -project TinyQuestion.xcodeproj -scheme TinyQuestion \
           -configuration Debug -derivedDataPath build build
open build/Build/Products/Debug/TinyQuestion.app
```

To produce a release DMG (universal arm64 + x86_64):

```sh
scripts/release.sh 0.1.0   # writes dist/TinyQuestion-0.1.0.dmg
```

`project.yml` is the source of truth for the Xcode project; the `.xcodeproj` is regenerated by `xcodegen` and is gitignored.
